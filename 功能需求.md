# 万能工具箱功能需求文档

## 1. 项目概述

### 1.1 项目背景
开发一套**一次开发、全终端复用、免频繁更新**的聚合工具箱，收拢日常开发与办公场景中的 IP 查询、时间转换、进制换算、科学计算器、单位转换等零散小工具。初期采用原生 HTML/CSS/JS 开发暴露了单文件协作难、依赖管理乱、多端打包复杂、版本更新依赖客户端重装等痛点，现基于 Vue3 + Vite 工程化重构，并引入四大工具分类架构与多容器打包方案，实现全端（Web/H5/小程序/鸿蒙/安卓/iOS/Windows/Linux/Mac）统一分发。

### 1.2 核心目标
- **全端复用**：一套 Vue3 源码，通过不同打包容器覆盖 Web、移动端、桌面端。
- **免更新扩容**：云端工具页面动态加载，新增工具无需客户端更新。
- **系统权限支持**：桌面端可调用串口、本地文件、硬件等系统级 API。
- **离线可用**：本地离线工具断网可用，云端离线工具缓存后可离线使用。
- **AI 协作友好**：组件化拆分，单工具修改仅交付对应组件，提升 AI 编码效率。

### 1.3 用户角色
- **普通用户**：使用工具箱完成计算、查询、转换等日常任务。
- **开发者**：扩展工具、配置分类、维护云端资源。
- **打包维护者**：针对不同平台执行打包与发布流程。

## 2. 系统总体架构

### 2.1 前端基座
- 基于 **Vue 3 + Vite** 构建，采用单文件组件（SFC）开发，目录标准化（公共组件、页面、样式、工具函数、依赖独立分层）。
- 使用 **Vue Router** 实现无刷新路由，首页分类→工具列表→工具页面平滑跳转。
- 全局自适应布局，一套页面适配手机、平板、桌面。

### 2.2 桌面端内嵌后端
- 首选方案：**Python Flask + PySide6（Qt）**。
- 桌面应用启动时自动启动 Flask 本地后端服务，前端通过 `localhost` HTTP 请求调用系统 API（串口、文件、硬件等）。
- Qt WebEngine 加载打包后的 Vue 静态资源，形成一体化桌面程序。
- 备选方案：Electron + Node.js（用于无硬件交互的轻量工具）。
- 打包目标：单 EXE 分发，无需预装 Python 或浏览器。

### 2.3 多端打包容器
- **Web / H5**：Vue 项目直接打包为静态资源，部署至服务器/CDN。
- **移动端**：Vue 源码导入 UniApp X 工程，编译生成安卓、iOS、鸿蒙、小程序、快应用。
- **桌面端**：核心方案为 Qt + Flask 打包；部分纯离线工具可选 Electron 打包。

## 3. 功能需求

### 3.1 四大工具分类与资源管理
工具箱以**页面存放位置（本地/云端）**和**运行是否需联网**两个维度划分四类工具，解决离线可用性与版本更新矛盾。

#### 3.1.1 本地离线工具
- **描述**：页面源码打包进安装包/本地资源，所有运算逻辑前端本地完成，无任何接口依赖，全场景断网可用。
- **典型工具**：计算器、进制转换、时间戳转换、单位换算等纯算法工具。
- **存储方案**：默认使用浏览器 LocalStorage / IndexedDB 前端持久化；桌面端需要大文件存储或硬件交互时可启用内嵌后端接管。
- **更新方式**：随客户端安装包更新。

#### 3.1.2 本地联网工具
- **描述**：页面内置，但功能依赖远端 API 接口，断网无法使用。
- **典型工具**：公网 IP 详细地理位置查询、实时汇率换算等。
- **存储方案**：配置信息本地前端缓存；业务数据依赖接口返回。
- **更新方式**：页面随客户端更新；后台接口独立迭代。

#### 3.1.3 云端离线工具
- **描述**：页面资源部署在自建服务器/CDN，首次使用需联网加载，加载后自动缓存至本地，之后可离线使用。
- **典型工具**：需要定期更新规则但运算可本地完成的工具（如离线文档查询、静态数据解析）。
- **缓存实现**：浏览器端使用 CacheStorage + LocalStorage 缓存静态资源和数据；桌面端可复用 Python 后端管理本地缓存目录，提供更灵活的控制。
- **更新方式**：云端部署新页面，客户端下次联网时拉取并刷新缓存，无需升级安装包。

#### 3.1.4 云端联网工具
- **描述**：页面托管云端，每次使用均需联网加载页面并请求后端接口，全程依赖网络。
- **典型工具**：实时行情、在线数据解析、协同编辑等。
- **更新方式**：云端直接部署，所有用户即时生效。

### 3.2 多终端分类视图自动适配
前端运行时自动识别当前环境（Web / 小程序 / 桌面客户端 / 移动 App），动态调整分类菜单展示：

- **移动端（安卓/iOS/鸿蒙/UniApp小程序）**：完整展示四大分类（本地离线、本地联网、云端离线、云端联网）。
    - 小程序特殊处理：无真正本地页面，原“本地分类”页面部署至小程序后台资源并支持自带缓存；“云端分类”使用自建 CDN。
- **桌面端（Windows/Linux/Mac）及 Web/H5**：合并为两大分类——**离线工具**（本地离线+云端离线）、**联网工具**（本地联网+云端联网），简化操作。

### 3.3 工具列表与动态加载
- 本地工具列表：由前端静态配置文件定义，随打包更新。
- 云端工具列表：客户端启动时异步请求 CDN 目录接口，获取可用云端工具清单，动态渲染至工具列表。
- 用户点击工具条目：直接进入 Vue Router 对应页面；若为云端页面，通过异步组件/远程组件机制加载。

### 3.4 内置默认工具
打包时内置以下本地离线工具，保证开箱即用：
1. **IP 查询**：支持本机 IP、公网 IP 查询（本地离线查询本机 IP，公网 IP 属于本地联网工具，视情况拆分）。
2. **时间戳/时区转换**：时间戳与日期互转、多时区换算，纯前端计算。
3. **科学计算器**：复杂数学运算、函数绘图等。
4. **多维度单位换算**：长度、面积、体积、重量、温度、汇率（汇率需要联网，归入本地联网）等，基础换算本地完成。

### 3.5 数据持久化与系统交互方案
系统预置三种数据存储与权限接入方式，统一抽象接口，工具可根据需要灵活配置：

#### 3.5.1 前端浏览器缓存（方案一）
- 适用：所有轻量工具，仅需少量键值或结构化数据存储，无需系统权限。
- 实现：封装统一 Storage 工具类，底层使用 LocalStorage（小数据）/ IndexedDB（大数据），全平台兼容。
- 特点：零依赖、即开即用。

#### 3.5.2 打包容器原生系统 API（方案二）
- 适用：打包为桌面客户端或移动端，需要访问本地磁盘、串口、硬件外设等系统能力。
- 实现：Electron / UniApp X 容器各自封装系统 API，前端通过统一 `platform` 适配层调用；桌面 Qt 方案则由 Python 后端提供相应 HTTP 接口。
- 平台差异：主体业务代码不变，平台独有逻辑单独封装。

#### 3.5.3 桌面端内嵌本地后端（方案三）
- 适用：Windows/Linux/Mac 桌面客户端，需大量文件读写、复杂硬件交互、数据库存储或未来可能拆分为远程服务。
- 实现：打包内置 Python Flask 后端，启动时自动在本机随机端口运行，前端通过本地 HTTP 请求 (`http://127.0.0.1:port`) 获取系统服务。
- 优势：后端生态完善，统一 API 屏蔽系统差异，前端仅需关心接口调用；后续转为远程服务仅需修改请求地址。

**工具存储模式切换**：每个工具可配置 `storageMode` 属性（`localCache` | `nativeAPI` | `localServer`），开发期默认用前端缓存，桌面正式版可按需切换。

### 3.6 离线缓存引擎
- 封装全局缓存管理模块，负责云端离线工具的页面静态资源缓存与版本控制。
- 首次加载云端页面时，自动将 HTML/JS/CSS 缓存至 CacheStorage 或桌面端指定缓存目录。
- 支持缓存过期策略（基于时间或版本号），联网时自动更新。
- 断网时自动拦截请求，返回缓存内容，保证离线可用。

### 3.7 环境识别与自适应
- 前端初始化时检测运行环境标识（通过 userAgent、平台注入变量或打包环境变量）。
- 根据环境：
    - 切换分类菜单形态。
    - 切换数据持久化接口实现（Web 环境仅允许方案一；桌面环境可使用方案一、二、三）。
    - 决定工具列表中显示哪些平台专属工具（如 Windows 注册表工具仅在 Windows 桌面端显示）。
- 响应式布局：使用 CSS media query 或 Vue 响应式组件适配不同屏幕尺寸。

### 3.8 平台专属工具扩展
- 为各平台独有功能设立专属目录（如 `tools/windows/regedit`）。
- 页面组件通过环境判断动态注册到工具列表，不影响其他平台打包体积和运行。
- 专属工具的数据存储默认绑定对应平台能力（如 Windows 注册表工具必然使用内嵌后端）。

### 3.9 工具开发与 AI 协作规范
- 每个工具独立为 Vue 单文件组件（或包含少量子组件的文件夹），公共逻辑抽取为全局组合式函数（Composables）。
- 开发迭代时仅向 AI 提供当前工具相关的 1~2 个组件源码，全局配置、路由、公共依赖保持不变。
- 工具注册：开发者在 `toolsRegistry.js` 中填写工具元信息（名称、描述、分类、存储模式、平台限制、路由路径等），框架自动完成路由挂载与列表渲染。
- 云端工具：额外提供远程组件加载配置，指定 CDN URL。

## 4. 非功能性需求

### 4.1 性能
- 桌面端应用冷启动时间不超过 3 秒（含 Flask 后端启动及前端加载）。
- 云端页面首次加载后缓存，二次访问离线秒开。
- 打包体积控制：桌面端单 EXE 不大于 80MB（Qt + Flask 方案），轻量 Electron 版本不大于 100MB。

### 4.2 可用性
- 离线环境下所有本地离线工具和已缓存的云端离线工具功能完整可用。
- 网络切换时工具状态不丢失，失败时给予友好提示。

### 4.3 可维护性
- 前后端代码分离，公共依赖统一管理，单个工具修改不影响其他工具。
- 云端工具管理后台提供工具目录编辑、版本发布功能（可选简化版：手动维护 JSON 文件上传 CDN）。

### 4.4 安全性
- 桌面内嵌后端仅监听 `127.0.0.1`，防止外部网络访问。
- Flask 接口需要校验请求来源为本地前端，可加入简单的 Token 或 Origin 验证。
- 对系统 API 调用进行权限控制和参数校验，避免误操作。

### 4.5 兼容性
- Vue 前端需同时兼容浏览器标准 DOM、Electron 渲染进程、Qt WebEngine、UniApp X WebView。
- 移动端打包后需通过相应平台审核（小程序、App Store 等，仅技术兼容，合规由发布者处理）。

## 5. 技术约束与选型说明

### 5.1 前端技术栈
- Vue 3 (Composition API)
- Vite 5+
- Vue Router 4+
- Pinia（按需引入，用于跨组件状态共享）
- Axios（HTTP 请求，统一拦截器）
- 纯 HTML 子页面（少量极简工具可保留原生实现，通过 iframe 或直接链接整合）

### 5.2 桌面端首选方案
- 后端：Python 3.10+，Flask
- GUI 容器：PySide6 / PyQt6（WebEngine 组件）
- 打包：PyInstaller（生成单 EXE，包含 Python 解释器、依赖、Vue 静态资源）
- 系统交互：pyserial（串口）、psutil（系统监控）、watchdog（文件系统事件）等常用库

### 5.3 移动端打包
- 基于 UniApp X，引入 Vue 3 代码，通过条件编译处理平台差异。

### 5.4 Web 部署
- 标准 Vite 构建产出，部署到 Nginx / CDN，开启 Gzip 压缩与合理的缓存策略。

## 6. 项目迭代规划

1. **第一阶段**：搭建 Vue3 前端基座，实现四大分类静态演示，完成内置四大基础工具（本地离线）。
2. **第二阶段**：实现云端工具动态加载、离线缓存引擎，验证云端离线分类。
3. **第三阶段**：桌面端 Qt + Flask 模板开发，打通前后端联调，完成系统 API 示例工具（如串口助手、文件管理器）。
4. **第四阶段**：多端适配与打包流程完善，输出各平台安装包；平台专属工具扩展。
5. **持续演进**：云端工具独立开发上线，无需发版；后端服务可逐步迁移至远程，实现工具云化。
