# 词条阅读器 (Entry Reader) 功能需求规格说明书 V2.0

## 1. 概述
构建一个纯前端的词条管理工具，支持无限层级树形结构、版本历史、回收站、多种视图模式、多格式导入导出、主题/语言切换。数据持久化在浏览器 `localStorage` 中，无需后端。

## 2. 技术栈与环境
- 框架：Vue 3（通过 CDN 引入）
- Excel 处理：SheetJS (xlsx) CDN
- 样式：CSS 自定义属性，响应式布局
- 存储：`localStorage`
- 国际化：前端字典映射
- 运行环境：现代浏览器（Chrome, Firefox, Edge, Safari）

## 3. 数据模型

### 3.1 词条对象 (Entry)
所有词条存储在一个扁平数组中，每个对象包含以下字段：

| 字段 | 类型 | 描述 |
|------|------|------|
| id | String | 唯一标识，使用 `e_` + 时间戳 + 随机字符串生成 |
| parentId | String \| null | 父级词条ID，根词条为 null |
| content | String | 词条文本内容 |
| version | Number | 当前版本号，初始为 1 |
| history | Array | 历史版本记录列表，见 3.2 |
| isDeleted | Boolean | 是否逻辑删除，默认 false |
| deletedAt | Number \| null | 删除时间戳（毫秒），未删除时为 null |
| createdAt | Number | 创建时间戳 |
| updatedAt | Number | 最后更新时间戳 |
| copyCount | Number | 文本复制次数，初始 0 |
| modifyCount | Number | 修改次数，初始 0 |

### 3.2 历史记录对象 (History Record)
| 字段 | 类型 | 描述 |
|------|------|------|
| content | String | 该版本对应的内容 |
| version | Number | 当前版本号，初始为 1 |
| timestamp | Number | 创建该版本的时间戳 |

**规则**：
- 初始创建时生成一条 version=1 的记录。
- 每次修改（内容变化）时生成一条新记录，版本号自增。
- 恢复历史版本时，本质是**新增**一条记录（内容=历史版本内容，版本号=当前版本号+1）。

## 4. 核心功能模块

### 4.1 数据存储与初始化
- **存储键**：`entry-reader-data`（词条数组 JSON），`entry-reader-settings`（用户设置 JSON）。
- **加载**：页面挂载时读取 `localStorage`，若无数据则为空数组。
- **保存**：每次增、删、改、复制文本后自动调用 `saveData()`，将整个 `allEntries` 序列化保存。

### 4.2 词条管理

#### 4.2.1 新增词条
- **入口**：
    - 工具栏“新增根词条”按钮（`parentId=null`）
    - 每个词条行内的“➕”按钮（新增子词条，`parentId=该词条id`）
    - 层级导航弹窗中的“在此新增”按钮
- **编辑框**：
    - 自定义模态框，包含多行文本域（`<textarea>`）和提示语“每行一个词条（可批量创建）”。
    - 提交时按换行符分割成多个词条，去除空白行。
    - 每个新词条生成唯一ID，设置 `createdAt` 和 `updatedAt` 为当前时间，`version=1`，并初始化 `history` 为一条记录。
- **批量创建**：一行一个词条，一次性添加多个子节点（或根节点）。

#### 4.2.2 编辑词条
- **入口**：词条行“✏️”按钮。
- **编辑框**：弹出模态框，预填当前内容，单行/多行均可，但编辑只对单个词条生效。
- **提交逻辑**：
    - 若新内容与旧内容**完全相同**，提示“内容未变化”，不更新任何数据。
    - 若内容变化：
        - `content` 更新
        - `version = version + 1`
        - `modifyCount = modifyCount + 1`
        - `updatedAt = 当前时间戳`
        - `history.push({ version: 新版本号, content: 新内容, timestamp: 当前时间戳 })`

#### 4.2.3 删除词条（逻辑删除）
- **入口**：词条行“🗑️”按钮。
- **确认**：弹出浏览器 `confirm` 对话框。
- **执行**：
    - 设置 `isDeleted = true`
    - `deletedAt = 当前时间戳`
    - `updatedAt = 当前时间戳`
- **不影响子节点**：子节点保持原有状态，但如果子节点未被删除，不会显示在主页树中（因为父节点已删除）。

#### 4.2.4 恢复词条
- **入口**：回收站中顶级被删除词条的“♻️”按钮。
- **条件**：该词条 `isDeleted === true`，且满足顶级条件（`parentId === null` 或父级未被删除）。
- **执行**：
    - 设置 `isDeleted = false`
    - `deletedAt = null`
    - `updatedAt = 当前时间戳`

#### 4.2.5 复制文本
- **入口**：词条行“📋”按钮。
- **动作**：将该词条的 `content` 写入系统剪贴板。
- **成功后**：该词条的 `copyCount` 加 1，立即保存。
- **注意**：不复制词条本身，只复制文本内容。

### 4.3 版本历史与恢复

#### 4.3.1 查看历史
- **入口**：词条行“📜”按钮。
- **弹窗**：显示当前词条 `history` 数组，按版本号排序。
- **展示**：
    - 每行：版本号、内容、时间。
    - 当前版本（`h.version === entry.version`）高亮背景，并显示“当前”标签，无恢复按钮。
    - 其他版本显示“恢复”按钮。

#### 4.3.2 恢复历史版本
- 点击历史版本的“恢复”按钮。
- **逻辑**：
    - 取该历史版本的 `content`
    - 执行类似编辑操作：版本号+1，修改次数+1，`history` 新增一条记录（版本号=新版本号，内容=历史内容，时间=当前时间）。
    - 实际上是**在前端创建一个新版本**，而不是回退覆盖。

### 4.4 回收站

#### 4.4.1 视图切换
- 工具栏“回收站”按钮，显示已删除词条数量。再次点击返回主页。

#### 4.4.2 展示规则（树形/层级导航模式）
- **回收站树形**：
    - 只显示**顶级被删除词条**（`isDeleted=true` 且父级未被删除或父级为空）。
    - 顶级词条展开时，递归显示其**所有子节点**（无论子节点是否被删除），以真实还原删除时的树状结构。
    - 子节点不显示恢复按钮，只有顶级被删除词条显示恢复按钮。
- **回收站平铺**：直接拉取所有 `isDeleted=true` 的词条，忽略层级，按排序规则展示。

#### 4.4.3 与主页的关系
- 主页和回收站共享同一份 `allEntries` 数据。视图变化仅通过筛选条件实现。
- 主页不显示任何 `isDeleted=true` 的词条（树形或层级导航模式下，若父节点删除，子节点即使未删除也不显示，因为是从根向下筛选）。

### 4.5 排序规则

所有词条列表（除手动搜索外）均按以下加权分数**降序排列**：

```
score = (1 / (1 + hoursDiff * 0.1)) * 10 + copyCount * 0.5 + modifyCount * 0.3
```
其中 `hoursDiff = (当前时间戳 - updatedAt) / 3600000`（若 `updatedAt` 不存在则用 `createdAt`）。

- 根级别、子级别均使用此排序。
- 搜索模式下，结果同样按此排序。

### 4.6 搜索与过滤

- 搜索框实时过滤（输入即触发）。
- **算法**：文本相似度（不区分大小写）：
    1. 若字符串 a 包含 b 或 b 包含 a，基础得分 = 0.7 + (minLen/maxLen)*0.3
    2. 否则计算编辑距离（Levenshtein），相似度 = 1 - distance / maxLen
    3. 阈值：相似度 > 0.25 视为匹配。
- **树形视图下的特殊处理**：搜索时自动展开包含匹配词条的祖先节点，使匹配项可见。

### 4.7 视图模式

#### 4.7.1 树形视图 (Tree)
- 默认视图。
- 每个词条行首有展开/折叠箭头（▶），仅当词条拥有任何子节点（包括已删除子节点，但主页只显示未删除的）时可点击。
- 点击箭头：展开/折叠直接子级（递归组件）。
- 点击词条内容：等同于点击箭头（切换展开/折叠）。
- 子级缩进：每层增加 22px 的 `padding-left`。

#### 4.7.2 层级导航视图 (Breadcrumb)
- 顶部显示面包屑导航栏：
    - “🏠 根”始终可点击（返回第一级）。
    - 当前路径各级可点击（回到对应层级），最后一级为当前层级，不可点击但加粗显示。
- 主列表：只显示当前层级下的子词条（若未进入任何层级则显示根词条）。
- 点击某个词条：进入该词条的下一级，面包屑增加一项。
- 面包屑项左侧显示“›”分隔符。
- **回收站**下同样适用。
- **新增**：工具栏的“新增根词条”仍可用；词条行的“➕”按钮新增到当前词条下。

#### 4.7.3 平铺视图 (Flat)
- 完全忽略层级结构。
- 主页：显示所有 `isDeleted=false` 的词条。
- 回收站：显示所有 `isDeleted=true` 的词条。
- 所有结果按排序规则排列。

## 5. 导入导出功能

### 5.1 支持的格式及规格

| 格式 | 导出结构 | 导入解析规则 |
|------|----------|--------------|
| JSON | `{ entries: [...] }`，每个条目包含全部字段及嵌套 `children` 数组（若导出仅活跃词条） | 递归扁平化，还原 `parentId` 和所有属性（含 `history`） |
| XML | `<entries><entry id=".." parentId=".." content=".." version=".." ... ><entry .../></entry></entries>` | 解析 DOM，递归提取属性及子节点，忽略 `history`（首次导入仅创建版本1） |
| TXT | 每行一个根词条内容 | 每行生成一个根词条（`parentId=null`），版本1 |
| YML | 用缩进表示层级，每行以 `- ` 开头 | 解析缩进，构建父子关系，内容为新词条 |
| Properties | 用点号分隔层级作为键，值为内容，如 `a.b.c=value` | 按点号拆分键，构建路径，若无中间词条则自动创建占位词条？**（实际实现：仅当路径中已有对应词条时建立父子关系，否则不创建中间节点。这里建议明确为：按照键的层级，若中间词条已存在于导入的上下文中则关联，否则不自动创建。）** |
| CSV | 首行字段：`id,parentId,content,version,copyCount,modifyCount,createdAt,updatedAt,isDeleted,deletedAt` | 按列解析，`id` 冲突时重新生成，映射 `parentId` |
| Excel | **两个工作表**：Sheet1 “Entries”（字段同 CSV），Sheet2 “History”（字段：`entryId, version, content, timestamp`） | 同时读取两个Sheet，先建立词条对象（重新生成ID），然后根据 History 中 `entryId`（旧ID）映射，将历史记录填充到对应新ID的词条中；若 Sheet2 缺失则词条使用默认版本1历史。|

**导入通用规则**：
- 所有导入条目会与现有数据合并，不会覆盖。
- 重新生成 ID 以保证唯一性，同时维护 `parentId` 映射表（旧ID→新ID）。
- 若导入格式不包含时间戳，则使用当前时间。
- Excel 导入是唯一可完整恢复历史版本的格式。

### 5.2 导出逻辑
- 导出仅包含**活跃词条**（`isDeleted=false`）。
- JSON/XML 导出时保留树形结构和 `history` 等完整数据。
- TXT 只导出根词条的内容。
- YML 按层级缩进导出内容。
- Properties 以内容构造路径（需保证内容不含 `=` 和 `.`，否则用下划线替换）。
- Excel 导出两个工作表，列顺序固定。

## 6. 用户界面

### 6.1 布局结构
```
┌─────────────────────────────────────────┐
│ 标题  回收站  新增  [视图] [主题] [语言] 导入/导出 │
├─────────────────────────────────────────┤
│ 搜索框    结果统计    总词条/活跃  │
├─────────────────────────────────────────┤ (层级导航时面包屑)
│ [面包屑: 🏠 根 › 一级 › 当前 ]         │
├─────────────────────────────────────────┤
│ 词条列表区（树/列表/平铺）            │
│ ...                                     │
├─────────────────────────────────────────┤
│ 空状态提示（无数据时）                 │
└─────────────────────────────────────────┘
```

### 6.2 词条行设计
- 左侧：展开箭头（树形模式）
- 中间：内容文本，单行省略
- 右侧标签组：版本号 vX，复制次数📋X，修改次数✏️X
- 操作按钮（默认隐藏，悬停显示；回收站模式下始终显示）：
    - 📋 复制文本
    - ➕ 新增子词条
    - ✏️ 编辑
    - 📜 历史版本
    - 🗑️ 删除
    - ♻️ 恢复（仅回收站顶级删除）

### 6.3 模态框
- 编辑/新增弹窗：标题、`<textarea>`（多行提示）、保存/取消按钮。
- 历史版本弹窗：历史列表，当前版本高亮+标签，恢复按钮。
- 导出格式菜单：小型弹窗，列出格式按钮。
- 所有弹窗背景遮罩，点击遮罩可关闭。

### 6.4 主题
- 通过 `[data-theme="dark"]` 选择器覆盖 CSS 变量实现深色模式。
- 支持浅色/深色切换，状态保存到 `localStorage`。

### 6.5 国际化
- 支持 zh-CN、zh-TW、en，通过字典 `i18n` 动态渲染。
- 语言选择存储在 `localStorage`。

## 7. 边界情况与异常处理
- 内容为空时不允许保存。
- 编辑时若内容未变不更新版本、不记录历史。
- 删除时通过 `confirm` 确认。
- 导入文件格式不支持时提示错误。
- Excel 导入时若 Sheet2 缺列或缺失，不影响主表导入（仅丢失历史）。
- 回收站中，若父级被删除但子级未删除，子级在回收站树形下仍显示（因为父级已删除，其子级作为该父级的后代展示，符合“删除级联可见”的逻辑），但不显示恢复按钮。
- 搜索时如果树形视图未展开，自动展开祖先。

## 8. 数据持久化
- 词条数据：`localStorage.setItem('entry-reader-data-v2', JSON.stringify(allEntries))`
- 设置数据：`localStorage.setItem('entry-reader-settings', JSON.stringify({theme, lang, viewMode}))`
- 每次数据变更后立即保存，无需手动保存。

## 9. 非功能性需求
- 操作响应流畅，视图切换即时。
- 导出文件命名包含日期。
- 所有时间戳以毫秒为单位存储和计算。
- 移动端响应式适配（@media 断点）。
