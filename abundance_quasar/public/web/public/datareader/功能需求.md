# 树形数据阅读器（Tree Data Reader）全量功能需求文档 v1.0

## 一、核心定位
开发一款跨端适配的**树形 / 表格结构数据阅读器**网页应用。支持 JSON、XML、YAML、Properties、Excel 等多种数据格式的导入、导出、可视化预览、Key 别名配置、值格式化（枚举/日期）等功能。界面风格参考代码编译器 / Notepad++，采用深色 / 浅色双主题，支持多文件标签管理、灵活布局切换、代码折叠及专属格式保存与恢复，满足复杂层级数据的高效阅读与配置管理需求。

---

## 二、文件与标签管理
### 1. 多文件支持
- 可同时打开多个文件，以标签页形式展示在顶部标签栏。
- 每个标签显示文件名与关闭按钮，当前激活标签高亮。
- 点击标签可切换文件，切换时自动保存当前文件编辑状态（内容、配置、布局）。
- 支持关闭标签，当仅剩一个文件时提示不可关闭。

### 2. 文件操作
- **新建**：创建空白 JSON 文件（预填示例数据），生成唯一文件 ID。
- **打开**：通过系统文件对话框选择本地文件，自动识别扩展名：
    - `.json / .txt` → JSON
    - `.xml` → XML
    - `.yml / .yaml` → YAML
    - `.properties` → Properties
    - `.xlsx / .xls` → Excel（自动解析为 JSON 对象，工作表名为键）
- **保存**：将当前编辑器内容保存为原文件（覆盖下载）。
- **另存为**：弹出格式选择框，可将当前数据转换为 **JSON / XML / YAML / Properties** 四种格式并下载，文件名可自定义。
- **导出Excel**，将层级阅读页展开的所有表格导出，每个表格作为一个单独的sheet。
- **导出专属格式 (.dat)**：将原始数据、格式类型、所有 Key 配置、忽略规则、显示模式、折叠状态、当前展开的表格层级路径、布局模式等打包为 JSON 文件下载。
- **导入专属格式 (.dat)**：加载 .dat 文件，完整还原上次保存时的编辑器内容、别名、配置、折叠与展开状态及布局。

## 三、界面布局与交互
### 1. 整体结构
顶部菜单栏 → 文件标签栏 → 内容区（含“数据预览”和“Key 配置”两个子标签页）。

### 2. 数据预览页布局切换
- 支持三种布局模式，通过**编辑器和阅读页标题栏右侧的布局按钮组**同步切换（避免仅阅读页模式下按钮消失）：
    - **左右分栏**：左侧编辑器 + 右侧级联阅读页
    - **仅编辑器**：全屏编辑器
    - **仅阅读页**：全屏阅读页
- 布局状态随文件保存，切换文件时恢复。

### 3. 响应式适配
- 使用 Flex 弹性布局，移动端（≤768px）自动切换为**上下布局**（编辑器在上，阅读页在下）。
- 按钮、单元格最小点击区域 44×44px，适配触屏。

## 四、编辑器功能
### 1. 语法高亮
- 根据当前数据格式（顶部下拉选择）提供不同的语法着色：
    - **JSON**：Key 浅蓝，字符串浅橙，数字浅绿，布尔/Null 蓝色，标点灰色。
    - **XML**：标签、属性、字符串分别着色。
    - **YAML**：Key、字符串、数字、布尔、Null、注释均有独立颜色。
    - **Properties**：Key 与值区分高亮，注释灰色。
- 高亮层覆盖在透明 textarea 之上，同步滚动。

### 2. 行号与代码折叠
- 编辑器左侧显示行号，自动跟随内容行数。
- 可折叠 / 展开代码块：
    - JSON：`{}` 和 `[]` 块，行号旁显示 `▾` / `▸` 按钮。
    - XML：标签块折叠。
- 折叠状态保存在文件对象中，随 .dat 导出 / 导入。

### 3. 实时同步
- 编辑器输入时实时更新行号、语法高亮，并防抖（400ms）解析数据更新阅读页表格。

## 五、层级阅读页（核心）
### 1. 级联表格逻辑
- 将解析后的数据逐层渲染为表格，每个表格代表一个嵌套层级。
- 根数据自动创建“层级 1”表格，并根据数据内容动态生成表头：
    - **对象类型**：键为列头，值为单元格，单行展示。
    - **数组类型**：
        - 若数组元素**全为纯对象**（非数组、非标量），则自动提取所有对象的键作为列头，每一行为一个元素，形成完整的数据表格。
        - 若数组元素为对象数组**全为对象**，则自动提取所有对象的键作为列头，每一行为一个对象，形成完整的数据表格。
        - 若数组元素的 `值` 为标量或混合类型，则以可展开的可点击链接样式展示单元格。
- 表头支持显示 **原始 Key** 或 **别名**（根据全局配置动态切换）。

### 2. 可展开单元格
- 单元格内容为**对象**或**数组**时，显示为可点击链接样式，可自由选择显示原内容，不显示原内容时用特殊格式代替（如 `{对象}` / `[数组: N]`），并带有类型标记。
- **点击**后，在当前层级表格下方追加新层级表格，展示该单元格内部数据的解析结果。
- 支持**多分支回溯**：保留所有历史层级，点击历史表格中的其他可展开单元格，会截断后续层级并重新展开新路径。
- 已展开的单元格颜色高亮，标识当前活跃路径。

### 3. 值格式化展示
- 根据 Key 配置中设定的字段类型，单元格内容按规则渲染：
    - **枚举**：配置了枚举映射（如 `1=启用,0=禁用`），则显示映射后的文本。
    - **日期**：配置了日期格式（如 `YYYY-MM-DD`），尝试将值（时间戳或日期字符串）格式化为指定格式展示。
    - 其他标量值保留原始类型颜色（字符串、数字、布尔、null）。

## 六、Key 配置页
### 1. 全局配置
- **命名忽略规则**：下拉选择匹配 Key 时是否忽略大小写、驼峰命名或下划线/横线（影响别名匹配逻辑和同数组内多对象的表格展示）。
- **表头显示模式**：切换阅读页表格表头显示“原始 Key”还是“别名”。

### 2. Key 列表与单项配置
- 自动解析当前数据中所有**完整路径 Key**（如 `settings.database.host`），去除重复并列出。
- 字典配置
    - 提供新增或移出**字典**的功能，每项以表单形式填写，一个枚举可以有多对枚举值。
- 每个 Key 可独立配置：
    - **别名**：自定义显示名称，修改后实时刷新表头。
    - **字段类型**：普通 / 枚举 / 日期。
        - 选择“枚举”时，显示“枚举映射”输入框（格式 `key=value, key=value`）或输入枚举的名称。
        - 选择“日期”时，显示“日期格式”输入框（如 `YYYY-MM-DD`、`YYYY年MM月DD日`）。
- 全局配置
    - 提供**一键生成别名**：自动将 Key 的末尾段转为可读别名（去除特殊字符，分割驼峰）。
    - 提供**清除所有别名**按钮。

## 七、跨端与国际化
### 1. 三语切换
- 支持**简体中文 / 繁體中文 / English**，右上角下拉切换。
- 所有按钮文字、标题、提示信息、占位符等同步翻译，切换时**完整替换文本，不会出现文字叠加**。
- 语言设置持久化，刷新不丢失。

### 2. 双主题
- 支持**暗色 / 亮色**主题，右上角一键切换。
- 背景、文字、边框、语法高亮、表格、输入框、滚动条等全部元素适配。
- 主题设置自动保存。

---

## 八、数据格式支持详情
- **JSON**：原生解析，支持格式化（缩进 2 空格）。
- **XML**：通过 `xml-js` 库解析与生成，支持紧凑格式。
- **YAML**：通过 `js-yaml` 库解析与生成。
- **Properties**：自行实现解析（支持嵌套键通过 `.` 分隔），导出时平铺为 `key=value`。
- **Excel**：通过 `xlsx` 库将工作簿转为 JSON 对象（每个工作表对应一个属性，值为二维数组）。
- **专属格式 .dat**：JSON 结构，包含版本号及上述所有状态信息。

---

## 九、交互与辅助功能
- **快捷键**：`Ctrl+Shift+F` 格式化当前文件，`Ctrl+S` 保存。
- **格式化按钮**：一键美化 JSON/XML/YAML 内容，非法格式弹出错误提示。
- **Toast 提示**：所有操作（保存、打开、导入、导出、错误）均有右下角浮动提示。
- **编辑器 placeholder**：提示用户输入或打开文件。
- **空状态提示**：阅读页无有效数据时显示引导文字。

---

## 十、技术实现要求
- 纯前端实现，不需要后端，使用HTML/CSS/JS前端三件套或使用vue3二选一，可以适当加载第三方库。
- 使用 Flex 布局 + 相对单位（vh/vw/rem）实现跨端适配。
- 支持主流现代浏览器（Chrome、Firefox、Edge、Safari）。
